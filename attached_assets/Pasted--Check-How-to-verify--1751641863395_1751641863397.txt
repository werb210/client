| # | Check                                           | How to verify                                                                                                                                                                                                                                                                                                                                                |
| - | ----------------------------------------------- | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------ |
| 1 | **Hard-refresh the client bundle** (cache-bust) | Open the browser devtools → Network tab → Disable cache → Refresh. Ensure `POST https://staffportal.replit.app/api/applications` appears in the log.                                                                                                                                                                                                         |
| 2 | **Compare the exact request URL**               | In the Network tab, right-click the failing request → “Copy -> Copy as cURL” and paste it into a terminal. It should succeed (201/200). If curl works but the browser shows 404, it’s a CORS mismatch.                                                                                                                                                       |
| 3 | **Look for trailing-slash or path drift**       | The successful curl from staff must match the client-side path, e.g.<br>`/api/applications`<br>`/api/upload/<id>`<br>`/api/applications/<id>/initiate-signing`<br>`/api/applications/<id>/submit`. Any extra segment in the browser log (such as `/applications/applications`) means the router path in Express or the fetch URL in the client is still off. |
