| What to check                                                              | Why it breaks                                                                                                                | How to fix                                                                                                                                                                                                                |   |                                             |
| -------------------------------------------------------------------------- | ---------------------------------------------------------------------------------------------------------------------------- | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- | - | ------------------------------------------- |
| **1 ∙ Build step actually ran** (`npm run build` creates a `dist/` folder) | If `dist/` is missing the server has nothing to serve, so Express (or the default Node runner) falls back to “Cannot GET /”. | In the Replit shell run:<br>`npm run build`<br>Confirm a fresh `dist/` (or `build/`) folder appears and contains `index.html`.                                                                                            |   |                                             |
| **2 ∙ Run command serves the build, *not* dev mode**                       | Running `vite` or `react-scripts start` in prod leaves no static handler on `/`; only hot-reload endpoints are exposed.      | In `.replit` (or *Shell ▶ Run button*) use **one** of:<br>`run = "npm run preview"`  ← Vite<br>`run = "serve -s dist"` (install `serve`)<br>`run = "node server/static.js"` (if you wrote a small Express static server). |   |                                             |
| **3 ∙ Replit port binding**                                                | Replit expects the web process to listen on **\$PORT** (passed in env).                                                      | In custom server code:<br>\`const PORT = process.env.PORT                                                                                                                                                                 |   | 4173;`<br>`app.listen(PORT, '0.0.0.0', …)\` |
| **4 ∙ Public path & HTML fallback**                                        | A SPA must route *every* unknown path back to `index.html`; otherwise the server replies “Cannot GET /step-2”.               | If you run Express:<br>`ts<br>app.use(express.static('dist'));<br>app.get('*', (_, res) => res.sendFile(__dirname + '/dist/index.html'));<br>`<br>`serve -s dist` already handles this.                                   |   |                                             |
| **5 ∙ DNS / Replit domain mapping**                                        | The domain is verified but not **routed** to the Repl.                                                                       | In the *“Custom Domains”* tab, ensure the domain shows **green “Connected”** (not just “Verified”). If not, copy the final CNAME/A record Replit gives and wait for propagation.                                          |   |                                             |
| **6 ∙ Check logs after hitting `/`**                                       | Replit’s “Console” will log the actual request route and which handler replied “Cannot GET /”.                               | Hit the domain in a new tab ⇒ observe log. If you see “\[express] GET / → 404” it means your static‐file middleware isn’t running.                                                                                        |   |                                             |
